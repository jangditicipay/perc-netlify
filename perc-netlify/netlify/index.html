PERC-26-07CRITICAL
Wholesale eCommerce Site Hit by Hidden Wrapper Code — Cards Stolen Silently at Checkout
🌐 Global
Attackers exploited a file upload feature to plant a hidden web shell on a wholesale eCommerce site, then modified the checkout page's payment form with secret wrapper code that silently copied full card details from every customer — completely invisible to shoppers and security tools alike.
MediumeCommerce Website — File Upload Feature
AttackPHP Web Shell + iframe Wrapper Code Skimmer
Issuer RiskMass PAN, CVV2, name, address and expiry exposure — high fraud liability and chargeback volume
Customer RiskFull card details stolen silently during checkout — no visible warning
Mar 2026
Digital SkimmerWeb ShelleCommerceiframe HijackIOCs Included
How the Attack Worked
📁
Step 1 · Entry
Exploited the site's file upload feature to plant a PHP web shell — giving hidden, persistent server access
🔧
Step 2 · Persistence
Web shell used to run commands on the server undetected, preparing for the main attack
🔀
Step 3 · Wrapper Injection
The legitimate checkout payment form (payment iframe) silently swapped for an attacker-controlled fake
🃏
Step 4 · Silent Skimming
Every customer's card number, name, address, expiry and CVV2 captured invisibly
📤
Step 5 · Exfiltration
Stolen data logged to a hidden file; attackers periodically return to collect it
Malware Indicators — Block These
Malicious filename (WSO Web Shell v2.5)
sw C7B9BE81.php
SHA256
5c4e158ce40c71fae9cfa2b1ac07ae9a6b90f2f13d7c5259974491af09c816c9
⚠ Scan upload directories for this file immediately
What to Do
  • Restrict file upload features to authenticated admin users only
  • Use File Integrity Monitoring to alert on any unauthorised changes to payment-related scripts
  • Monitor web server logs for unexpected PHP files in upload directories
  • Banks: use CAMS alerts to block compromised PANs from skimming incidents
